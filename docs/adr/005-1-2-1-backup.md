# ADR-005: 1-2-1 backup strategy (S3 + Yandex Disk)

## Статус
Accepted (supersedes 3-2-1)

## Контекст
Правило 3-2-1 (3 копии, 2 носителя, 1 offsite) — индустриальный стандарт.
Однако VPS на cloud.ru free-tier имеет всего 30 ГБ SSD — локальная копия restic занимает
ценное место и не добавляет отказоустойчивости (диск всё равно один).

На VPS критичны: конфиги (/etc), данные приложений (/var/www), пользовательские файлы (/home).

Требования:
- Автоматический daily backup
- Encrypted at rest (AES-256)
- Deduplication (экономия места)
- Snapshot-based (точечное восстановление)
- Минимум зависимостей
- **Не занимать место на SSD**

## Решение
**Инструмент: restic** — AES-256-GCM, dedup, snapshots, rclone для Yandex Disk, s3 backend.

**Две копии (1-2-1):**
1. **cloud.ru S3** — restic s3 backend, AES-256, 15 ГБ free-tier
2. **Yandex Disk (offsite)** — restic + rclone, географически отдельный дата-центр

Локальная копия на SSD удалена для экономии дискового пространства.

**Retention:** 7 daily / 4 weekly / 3 monthly / 1 yearly (настраивается в `backup/config.yaml`)

**Расписание:** cron `0 2 * * *` (ночь, минимальная нагрузка)

**Конфигурация** (`backup/config.yaml`):
```yaml
sources:
  - /etc
  - /home
  - /var/www
  - /var/log
  - /root
retention:
  keep_daily: 7
  keep_weekly: 4
  keep_monthly: 3
  keep_yearly: 1
```

**Команды:** `create`, `list`, `restore`, `setup`, `status` (все через `backup/backup.py`)

## Альтернативы
- **3-2-1 с локальной копией**: занимает 500+ МБ на SSD, бесполезно при отказе диска
- **Borg**: медленнее restic на больших данных, нет встроенного S3/s3 поддержки
- **rsync + сценарии**: не snapshot-based, нет dedup, сложный restore
- **Только S3**: single point of failure при проблемах с провайдером

## Последствия
- Ежедневный backup: ~8 MB/day (сжатие + dedup, зависит от изменений)
- Восстановление: `python3 backup/backup.py restore --source s3`
- Все копии зашифрованы одним `restic/password` из BSM/.env
- Очистка старых snapshot-ов автоматическая (`forget --prune`)
- Cron настроен через `backup/backup.py setup` (добавляет строку в crontab)
- Лог backup: `/var/log/backup.log`
- 30 ГБ SSD полностью доступны для приложений
